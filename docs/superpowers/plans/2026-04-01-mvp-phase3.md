# MVP Phase 3: AtomicSkill Level Tracking + Recommendation Upgrade

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Track student AtomicSkill levels as they clear study sessions, surface skill growth on the EvalPage, and upgrade the recommendation engine to surface nodes whose prerequisite skills are ready.

**Architecture:** A new `student_skill_levels` table tracks each student's proficiency (0–3) per AtomicSkill. On every cleared session submission, the backend walks the graph's `teaches` edges for that node and increments the student's level for each taught skill. The frontend EvalPage fetches the updated levels after clearing and displays a "스킬이 자랐어요" card section. The recommendation engine gains a new scoring rule: nodes whose required AtomicSkills are all at level ≥ 1 receive a score boost and the reason "선수 스킬 준비됐어요".

**Tech Stack:** FastAPI/SQLite backend, React 18/TypeScript frontend, existing `authFetch`, `API_BASE = import.meta.env.VITE_API_URL || '/api'` pattern.

**Key schema note:** The `edges` table columns are `graph_version_id`, `id`, `edge_type`, `source`, `target`, `note`. Edge types in the graph are `teaches` (CS→AS) and `requires_skill` (AS→CS). When querying teaches edges: `edge_type='teaches'` and `source=<nodeId>`. When querying required skills for a node: `edge_type='requires_skill'` and `target=<nodeId>`.

## AtomicSkill Reference

| ID | Label |
|----|-------|
| `AS.NUMBER_SENSE` | 수 세기와 수의 크기 비교 |
| `AS.PLACE_VALUE` | 자릿값 이해 (일, 십, 백, 천) |
| `AS.ADD_SUB` | 덧셈과 뺄셈 |
| `AS.MUL_DIV` | 곱셈과 나눗셈 |
| `AS.FRAC_BASIC` | 분수 개념과 크기 비교 |
| `AS.DECIMAL` | 소수 개념과 자릿값 |
| `AS.RATIO` | 비율과 비례 |

## Level Scale

| Level | Meaning | Threshold |
|-------|---------|-----------|
| 0 | unknown | no cleared sessions for any CS that teaches this skill |
| 1 | introduced | 1 cleared session for a CS that teaches this skill |
| 2 | practiced | 2 cleared sessions |
| 3 | mastered | 3+ cleared sessions |

```python
def _update_skill_levels(conn, user_id: str, node_id: str) -> None:
    """Increment skill levels for all AtomicSkills taught by node_id (up to max 3).

    Looks up edges with edge_type='teaches' and source=node_id in the published
    graph version. For each taught skill (target), upserts student_skill_levels.
    """
    from datetime import datetime, timezone

    # Find the published graph version id
    version_row = conn.execute(
        "SELECT id FROM graph_versions WHERE status='published' ORDER BY created_at DESC LIMIT 1"
    ).fetchone()
    if not version_row:
        return

    graph_version_id = version_row["id"]

    # Find all skills taught by this node
    taught_rows = conn.execute(
        """
        SELECT target FROM edges
        WHERE graph_version_id = ?
          AND edge_type = 'teaches'
          AND source = ?
        """,
        (graph_version_id, node_id),
    ).fetchall()

    if not taught_rows:
        return

    now = datetime.now(timezone.utc).isoformat()

    for row in taught_rows:
        skill_id = row["target"]
        existing = conn.execute(
            "SELECT level FROM student_skill_levels WHERE user_id=? AND skill_id=?",
            (user_id, skill_id),
        ).fetchone()

        if existing is None:
            conn.execute(
                "INSERT INTO student_skill_levels (user_id, skill_id, level, updated_at) VALUES (?,?,?,?)",
                (user_id, skill_id, 1, now),
            )
        elif existing["level"] < 3:
            conn.execute(
                "UPDATE student_skill_levels SET level=?, updated_at=? WHERE user_id=? AND skill_id=?",
                (existing["level"] + 1, now, user_id, skill_id),
            )
        # If already level 3, no-op
```

```typescript
import { authFetch } from '../auth/api';

const API_BASE = import.meta.env.VITE_API_URL || '/api';

export interface SkillLevelsResponse {
  levels: Record<string, number>;
}

/**
 * Fetches the current student's AtomicSkill levels.
 * Returns a map of skillId → level (0–3). Only skills with level > 0 are included.
 */
export async function fetchSkillLevels(): Promise<Record<string, number>> {
  const res = await authFetch(`${API_BASE}/skill-levels`);
  if (!res.ok) {
    throw new Error(`fetchSkillLevels failed: ${res.status}`);
  }
  const data: SkillLevelsResponse = await res.json();
  return data.levels;
}
```

## Implementation Notes

**Edge table columns to remember:**
- `graph_version_id`, `id`, `edge_type`, `source`, `target`, `note`
- `edge_type='teaches'`, `source=<CS node>`, `target=<AS skill>`
- `edge_type='requires_skill'`, `source=<AS skill>`, `target=<CS node>`

**Frontend API pattern:**
```typescript
const API_BASE = import.meta.env.VITE_API_URL || '/api';
// Always use authFetch from '../lib/auth/api' for authenticated calls
```

**Graph versions:** Always query the published version (`WHERE status='published' ORDER BY created_at DESC LIMIT 1`) before querying edges.

**Upsert pattern for skill levels:**
```python
# Check existing → INSERT if none, UPDATE if level < 3
```

**Level semantics summary:**
- Level increments by 1 each time a CS node that teaches the skill is cleared.
- Cap at 3 (never `level > 3`).
- `GET /skill-levels` only returns skills with `level > 0`.
- The recommendation reason "선수 스킬 준비됐어요" requires ALL required skills at level ≥ 1.
