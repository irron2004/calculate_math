본 문서는 포렌식 엔지니어의 관점으로, “개념 결함을 추적·수리하고 안전하게 상위 개념으로 비약시키는” 교습·커리큘럼 가이드라인을 실전 문서 형식으로 정리한 것입니다. 초2를 시작점으로 S1(초2 표현) → S2(고학년 브리지) → S3(최상위 문제) 흐름을 전제로 합니다.

---

## (1) 혼합 분석 — GI(천재적 통찰) × PR(문제 재정의) 〔1,500자+〕

### 1-1. GI = (O × C × P × S) / (A + B)

* O(관찰, 9/10): 시중 교재는 학년-단원 선형 진도 중심이라 전이가 약합니다. 반면 본 프로젝트의 철학은 덧셈→반복덧셈→등차→일차→변화율(미분 직관)을 하나의 구조 언어로 가르칩니다. 저연령에게도 안전하게 상위 개념을 ‘낯설지 않게’ 보여줄 수 있습니다.
* C(연결, 9/10): “차분-누적” 이중 렌즈가 모든 영역(수·대수·기하·확통·해석)을 관통합니다. 등차의 1차 차분 상수 ↔ 일차함수 기울기 상수 ↔ 평균변화율·순간변화율로의 자연스러운 도약이 가능합니다.
* P(패턴, 9/10): (i) 차분 상수/선형성, (ii) 누적합/면적, (iii) 비율/정비례—이 3축이 고교·경시의 핵심 골격입니다. 초등 표현으로도 패턴 발견→언어화→공식화 전이가 됩니다.
* S(종합, 8/10): 지식그래프(JSON)로 노드(개념)와 간선(선수·전이)을 운용하고, 노드마다 결함 모드·교정 드릴·전이 과제를 탑재합니다.
* A(고정관념, 3/10)/B(편향, 2/10): “학년=상한” 고정관념을 버리고, LRC(Leap Ready Criteria)로 과속만 통제하면 됩니다.

> GI 결론: 구조-전이 중심 커리큘럼은 낮은 학년으로도 높은 구조를 안전하게 습득시키는 최적 해법입니다.

### 1-2. PR(문제 재정의) = P₀ × T(θ) × S(φ) × M(ψ)

* P₀: “학년 진도에 맞춰 가르친다” → 실제 문제: “상위 개념 실패의 원인은 하위 미시 스킬 결함(자리올림, 차이 읽기, 표→식 전환)이다.”
* T(180° 관점 회전): 학년→개념이 아니라 개념→학년. 상위 개념(등차·일차)을 덧셈 언어로 재서술하여 내려보내기.
* S(0.1×~10× 범위 조정): 0.1×(나노 스킬): ‘차이 읽기 1칸·2칸’ / 10×(거시): ‘등차→일차→변화율’ 맥락 연결.
* M(+2 메타 이동): 교과서 → 지식그래프 엔진. 노드별 증거 문제·오개념·교정 루틴·전이 과제를 표준화.

> PR 결론: “무엇을 먼저 가르칠까(학년)?”가 아니라, “어떤 결함을 어떤 순서로 수리해 상위 구조로 안전 도약시킬까(개념 스레드)?”를 설계합니다.

## (2) 교습 방법 SOP(표준 운영 절차)

### 2-1. 수업 구조(45분/60분 공용) — 수업 악보(Score)

1. 도입(5′): 오늘의 구조 렌즈 선언(🔺차분 / ⬛누적). 생활 맥락 예(계단, 저금).
2. 탐구1(10′): CPA 중 C→P. 조작물(칩·수직선)로 일정 증가를 몸/눈으로 체험.
3. 탐구2(10′): P→A. 표→식 전환, (a_n=a_1+(n-1)d)를 말로 먼저 쓰게 함(Explain-first).
4. 전이 과업(10′): 같은 구조를 다른 맥락(생활·데이터·도형)으로 풀어 보게 함(동형 3형제).
5. 메타인지(5′): 오늘 구조를 자기 언어로 서술(키워드: 시작값/차이/기울기/누적).
6. 퀵 체크(5′): 6문항 타임어택(미시 스킬). LRC 지표(정확·반응시간·설명 루브릭) 측정.

> 팁: S1에서는 C·P 비율을 높이고, S2에서 P·A 균형, S3에서는 A 중심 + 구조 설명을 요구.

### 2-2. 교사 발문 스크립트(예시)

* 발견 유도: “연속 두 항의 차이는 항상 얼마였지?” “표의 열을 바꾸면 규칙이 더 잘 보일까?”
* 전이 유도: “이 저금 표도 ‘같은 만큼 늘어남’이 보이니?” “좌표에 찍으면 어떤 모양이 나오지?”
* 오개념 교정: “공차는 ‘항의 값’이 아니라 ‘항 사이의 차이’야. 예로 5,8,11에서 공차는?”
* 설명 촉진: “공식을 먼저 쓰지 말고, 말로 설명해 볼래? 시작값이 2이고 매번 3씩 늘면 5번째는 왜 14일까?”

### 2-3. 그룹 운영

* 군집화: 같은 결함 모드를 가진 학생끼리 스몰그룹(3–4명).
* 역할 분담: ‘표 작성자’–‘그래프 찍기’–‘설명가’–‘검증가’.
* 협력 루틴: Check–Mate 시트로 서로의 설명을 체크(핵심어 3개 이상 포함? 반례 제시?).

### 2-4. 평가·피드백

* LRC 게이트: 정확≥90% + 반응시간 백분위≥60 + 설명 루브릭≥3/4 → 다음 스텝(S2/S3) 개방.
* 오답 처방: 미시 스킬 1분 드릴(자리올림, 차이 읽기, 표→식)로 즉시 교정 → 동일 구조 다른 맥락 재도전.
* 부모 리포트: “이번 주 전이 성공 장면” 스크린샷 + 다음 주 LRC 목표 1줄.

### 2-5. 수업 안전 장치

* 과속 방지: LRC 미충족 시 스텝 유지 + 전이 맥락만 바꿔 재학습.
* 정서 케어: 실패 피드백은 과정 언어 사용(“차이를 먼저 찾자” vs “틀렸다”).
* 용어 관리: 초2에겐 “같은 만큼 더하기”, “뛰는 칸수”; 고학년 브리지에서 “공차”, “기울기” 도입.

## (3) 커리큘럼 작성 가이드(Writer’s Guide)

### 3-1. 유닛 구성 규격

* Unit = 6p: S1(2p: 조작·그림) + S2(2p: 표→식→그래프) + S3(1p: 최상위 구조 문제) + Review(1p: 전이 3세트 + 오개념 드릴).
* 문항 수: S1 8–10, S2 8–10, S3 4, Review 6.
* 아이콘: 🔺(차분), ⬛(누적), 🧩(전이), 🗣(설명). 페이지 상단 고정.

### 3-2. 문항 설계 원칙(“구조→표현→맥락”)

1) 구조 정의 → 2) 표현 선택(C/P/A 2쌍) → 3) 맥락 스킨(생활·데이터·도형) → 4) 설명 우선(말로 쓰기) → 5) 오개념 방지(⚠️라벨) → 6) 전이 세트(다른 맥락 2–3개).

### 3-3. 태그 표준(필수)

`concept_node, step(S1/S2/S3), lens(diff/acc), representation(C/P/A), micro_skill, context, cognitive_demand, difficulty_est(0~1), time_s, misconception, LRC_gate, source_ref(영감), licensing(original)`

예시

```json
{
  "id": "ALG-AP-S1-007",
  "title": "한 번에 두 칸씩 점프!",
  "concept_node": "ALG6:등차수열",
  "step": "S1",
  "lens": ["difference"],
  "representation": "C->P",
  "micro_skill": ["차이 읽기", "수직선 점프"],
  "context": "계단놀이",
  "difficulty_est": 0.35,
  "time_s": 60,
  "misconception": ["값과 차이 혼동"],
  "LRC_gate": "S2_open_if(acc>=0.9 && rt_pct>=0.6)",
  "source_ref": "inspired_structure_only",
  "licensing": "original"
}
```

### 3-4. 탑다운 × 바텀투탑 병행 절차

* 탑다운(백캐스팅): 수능·경시 구조 아틀라스를 먼저 만든 뒤(문장·수치 복제 금지), 각 구조를 초2 표현으로 내려보냄.
* 바텀투탑(프로토타입): 초2 S1 빠른 파일럿으로 오답·설명 로그를 수집 → 구조 아틀라스 보완.
* 동기화 회의(격주): 작가·교사·분석가가 오답 히트맵을 보고 유닛을 재배열.

### 3-5. 난이도·품질 보정

* 앵커 문항: 유닛마다 2–3개 고정 배치 → 판(에디션) 간 난이도 동등화.
* LRC 기준 관리: 초판은 보수적(정확 90%)으로 시작, 데이터 보고 85~92% 범위 튜닝.
* 편집 가이드: 숫자·단위·좌표축 표기 일관. 그래프는 격자·눈금 최소화(인지 부하 감소).

## (4) 천재적 아이디어 12가지(교습·집필 특화, 3,000자+)

1) 동형 3형제(Isomorphic Triplets) 패키지 — 한 구조(등차 차분 상수)를 생활·데이터·도형 3맥락으로 나란히 제시. 공통 질문/공통 문장 칸 포함.

2) Lens 전환 버튼(🔺↔⬛) — 같은 상황을 차분(증가량)과 누적(합) 렌즈로 왕복. “차이를 먼저 보고, 그 차이의 누적을 면적으로 본다.”

3) Explain-First & 반례(Contrasting Cases) — 정답보다 먼저 설명을 요구, 옆에 차분이 일정하지 않은 반례 표 제공.

4) Micro-Drill 60s 라이브러리 — 자리올림, 차이 읽기, 표→식 변환 등 원자 스킬 집중 훈련. 초/중/고 공통 스킬 태깅.

5) Backwards Bridge — 일차함수 그래프를 먼저 보여 주고 등차 표로 ‘해체’. 상·하향 전이 강화.

6) Teacher Score × 판서 레이아웃 — “표→식→그래프” 3구역 고정 배치로 수업 일관성·시선 흐름 관리.

7) 전이 모의고사(Structure-Only Test) — 동형 3형제 2세트 + 낯선 맥락 2세트로 전이 진짜 여부 측정.

8) 오개념 경고 라벨 & 즉시 처방 — 빈출 오류 하단 고정 + 동일 구조의 수치만 바꾼 1분 드릴.

9) 부모 코치 카드(10문장 스크립트) — 질문 10·칭찬 5로 가정 피드백 질 향상.

10) 앵커 문항 & 난이도 캘리브레이션 — 판간 난이도 등가, 정확/반응시간/설명 로그.

11) 도메인 확장 브리지(등차→확통/기하) — 등차 막대 면적 합을 통계·기하로 연결, “합=면적” 직관.

12) 작가–교사–분석가 3각 리뷰 루프 — 2주 주기로 오답 히트맵 공유, 텍스트/수치/컨텍스트/발문/판서/기준 튜닝.

## (5) 바로 쓰는 템플릿 & 체크리스트

### 5-1. 교수-학습 지도안(한 차시) 템플릿

단원/개념: 등차수열(차분 상수)

학습목표: “같은 만큼 늘어남”을 표·그래프로 설명하고, (a_n=a_1+(n-1)d)를 말로 서술할 수 있다.

준비물: 칩/수직선 보드, 좌표 격자, 스티커.

흐름(45′)

* 도입(5′): 🔺차분 렌즈 소개, 생활 사례 1개.
* 탐구1(10′): C→P 조작(칩 2개 시작, +3씩). 표 채우기.
* 탐구2(10′): P→A 전환, 일반항 말로 쓰기 → 식으로 옮기기.
* 전이(10′): 동형 3형제(용돈/막대/센서 데이터).
* 메타인지(5′): 키워드 3개로 한 문장(시작값/차이/기울기).
* 퀵체크(5′): 6문항 드릴 + LRC 확인.

평가/과제: 전이 모의고사 6문항(숙제), 부모 코치 카드 안내.

### 5-2. 유닛 집필 체크리스트

* [ ] S1/S2/S3 페이지 레이아웃 준수(2-2-1-1p)
* [ ] 동형 3형제 포함(생활·데이터·도형)
* [ ] Explain-first 칸과 반례 카드 포함
* [ ] 오개념 라벨 + 1분 드릴 3문항
* [ ] 앵커 문항 2–3개
* [ ] 태그(lens, representation, micro_skill, difficulty_est …) 완비
* [ ] LRC 기준 문구 삽입
* [ ] 숫자·좌표 표기 일관성 확인

### 5-3. S1→S2→S3 예시(요약)

* S1(초2): “칩 3개씩 더하면 다섯 번째는?” 표 채우고 차이 말로 말하기.
* S2(브리지): 표→식 전환, ((n,a_n)) 좌표 찍기, 기울기=공차 확인.
* S3(최상위): “계단 높이 합”을 두 방식(일반항 합 vs 그래프 사다리꼴 근사)으로 비교 설명.

---

핵심은 “차분(변화)과 누적(합)이라는 두 렌즈로 개념을 포착하게 하고, 계산보다 설명을 먼저 쓰게 하는 것”입니다. 위 SOP와 Writer’s Guide, 템플릿·체크리스트만으로도 바로 집필/수업을 시작할 수 있습니다. 요청 시 이 가이드를 작성용 문서(구글독/노션) 템플릿 또는 Mermaid/CSV로 변환하고, ‘덧셈→등차’ 유닛 6p 샘플도 제공합니다.
