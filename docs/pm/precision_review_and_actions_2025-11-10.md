## 2) 시니어 피드백 15항목 — 현 상태 판정 & 대책

등급: ✅해결 / 🟡보완 / 🔴미흡 (근거/조치 포함)

1. 문서 불일치: ✅ — SSOT 확정, 스펙 역할 분리 → 조치: `CHANGELOG`, `DECISIONS` 연계 유지
2. 핵심 의사결정 미반영: ✅ — PRD/Spec 반영 완료 → 조치: 서버 상수화/교사 모드 ANY 플래그
3. 타깃·JTBD 불명확: ✅ — Phase‑1 초2~초4 → 조치: 온보딩 토글 표기 강화
4. UX 플로우(세션 종료·보스 실패): 🟡 → 조치: PRD에 SessionEnd UX 명시(토스트/자동 포커스/리미디얼)
5. 문항 엔진 MQG: 🟡 → 조치: Spec에 MQG‑P1(중복/유일해/제약/Invariant/로그) + DoD 편입
6. 설명 채점 현실성: ✅ → 조치: 키워드 사전 부록 고정
7. 접근성 vs 그래프: ✅ → 조치: 리스트 기본 진입 A/B, 마이크로카피 추가
8. 수익화 구체성: 🟡 → 조치: 전환 모달 타이밍/문구 PRD 고정(“S2 두 번째 완료 직후”), S2 일부 무료 지속
9. 데이터 파이프라인 리스크: ✅ → 조치: CI 실패 메시지/롤백 절차 문서화
10. 로드맵 낙관성: ✅ → 조치: W3–W4 게이트(템플릿 30+MQG‑P1)
11. 법적 리스크: 🟡 → 조치: 라이선스 가이드/로그 문서화 착수
12. 경쟁 분석: ✅(플랜) → 조치: 2주 내 1차 리포트
13. 사용자 리서치: ✅(플랜) → 조치: 실행/PRD 반영
14. IRT/Rasch: 🟡 → 조치: 앵커 문항 노출량 기준·로그 포맷 부록화
15. A/B 인프라: 🟡 → 조치: 실험 토글/샘플사이즈/분석 파이프 비교표 추가

## 5) 쟁점에 대한 부분 반박/설명
- 보스 합격선 80 vs 90: Phase‑1 초기 정착을 위해 80/100 유지, 대신 S2 LRC 90%+즉시 재도전 루프 강화(Phase‑2 상향 검토)
- 전이 vs 유료화: Free( S1 무제한 + S2 일부 )로 전이의 ‘맛’ 제공, 보스/모의고사 등 고가치 경험은 유료로 차별화
