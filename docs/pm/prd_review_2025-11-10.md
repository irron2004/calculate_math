# PRD 기획자 검토 — 디아블로식 스킬트리 기반 연산 학습 서비스 (v1.0.1)

## 검토 요약 (Executive Summary)

**검토 대상**: `docs/pm/prd_skill_tree_v1.0.md` (v1.0.1)
**검토일**: 2025-11-10
**총평**: ⭐️⭐️⭐️⭐️ (4/5)

**한 줄 평가**: 이전 피드백을 충실히 반영하여 **실행 가능한 PRD**로 발전했으나, **몇 가지 모호한 부분과 누락된 세부사항**이 여전히 남아 있음.

---

## ✅ 주요 개선 사항 (이전 대비)

### 1. 문서 일관성 확립
- ✓ **Single Source of Truth** 명시: 본 문서가 메인 PRD임을 명확히 함
- ✓ 변경 요약(v1.0.1) 상단 배치: 무엇이 바뀌었는지 즉시 파악 가능
- ✓ DECISIONS.md 연계: 핵심 의사결정이 별도 문서에 기록됨

### 2. Phase 1 범위 현실화
- ✓ 타겟: "초2~중등" → **"초2~초4"**로 명확화
- ✓ 코스: 12개 → **3개** (덧셈/곱셈/분수)
- ✓ 스텝: S1/S2/S3 → **S1/S2만**
- ✓ 템플릿: 100개 → **30개**
- ✓ 일정: 6주 → **10~12주**

### 3. 핵심 의사결정 확정
- ✓ Unlock 규칙: ALL (Phase 1), ANY (교사 모드 + Phase 2)
- ✓ 보스전: 합격 80/100, 쿨다운 없음
- ✓ 노드 노출: 기본 흐림, 교사/부모 스위치
- ✓ 설명 채점: S1/S2 선택, S3 필수 (키워드 매칭)
- ✓ 접근성: 리스트 뷰 추가

### 4. UX 플로우 보강
- ✓ **SessionEnd UX** 섹션 추가 (130~135줄)
  - 성공: 해금 토스트 → 자동 포커스 → [바로 시작]
  - 보스 실패: AS 하이라이트 + 리미디얼 CS 2개 추천

### 5. 수익화 구체화
- ✓ 무료: S1 무제한 + S2 일부(3개)
- ✓ 유료: 전체 + 부모 리포트 + 교사 콘솔
- ✓ 전환 타이밍: S2 2번째 완료 시 모달

## 🟡 여전히 개선 필요한 부분

### 1. 비전과 실제 Phase 1 범위의 괴리 ⚠️

**문제**:
- **섹션 0 (한 줄 정의)**: "같은 구조를 다른 맥락에서 반복 노출해 **전이**를 가속"
- **섹션 2 (JTBD)**: "덧셈만 잘해도 **등차수열**이 보이게 해줘"
- **섹션 4 (커리큘럼)**: "12개 코스 × S1/S2/S3 = 36 노드", "AS 60~80개"
- **BUT, Phase 1 실제 범위**: "코스 3개 × S1/S2, 템플릿 30개"

**괴리**:
- PRD 전반부(섹션 0~4)는 **전체 비전**(12개 코스, 등차수열, 일차함수 등)을 설명
- Phase 1 범위(섹션 12)는 **덧셈/곱셈/분수 S1/S2만**
- 독자가 혼란스러울 수 있음: "등차수열이 Phase 1에 포함되나?"

**권장 조치**:
```markdown
# 2. 대상 사용자 & JTBD

## Phase 1 (초2~초4)
* **학생**: "덧셈/곱셈/분수를 **구조**로 이해하고 싶다"
* **학부모**: "아이의 **연산 실력 향상**을 눈으로 확인하고 싶다"

## Phase 2+ (초5~중등)
* **학생**: "덧셈만 잘해도 **등차수열**이 보이게 해줘"
* (등차, 일차함수, 평균변화율 등 전이 확장)
```

**우선순위**: P1 (1주 내)
