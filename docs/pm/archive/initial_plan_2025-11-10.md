좋아요. “초2 시작 → 한 개념당 S1(초2 표현) → S2(고학년 브리지) → S3(최상위 문제)”라는 철학을 지키면서, 시중 문제집·수능·경시 문제를 **구조(패턴)만 추출**해 재저작하는 **교재 제작 절차**를 아래처럼 설계해 드립니다.
아래 구성은 **분석(GI × CS 혼합, 1,500자+) → 실행 절차(체크리스트 포함) → 태그/그래프 규격 → 저작권·소싱 가이드 → 유닛 템플릿 → 샘플 유닛 → 천재적 아이디어 10개+(3,000자+) → 90일 일정** 순서입니다.

---

## 1) 혼합 분석: GI(천재적 통찰) × CS(복잡성 해결 매트릭스)

### 1-1. GI = (O × C × P × S) / (A + B)

* **Observation(관찰)** 9/10: 시중 교재는 학년·단원 선형 진도 중심, *전이(같은 구조의 다른 맥락)* 설계가 약합니다. 초저연령에 맞춘 **CPA(구체–그림–추상)**와 **개념 스레드(덧셈→등차→일차→변화율)** 설계를 결합하면 차별화가 큽니다.
* **Connection(연결)** 9/10: “덧셈의 반복 = 등차 항 생성 = 1차 차분 상수 = 일차함수 기울기 일정”으로 관통시키는 연결은 교육적 설득력이 높고, 수능/경시의 본질 패턴(차분·누적·비율)을 한 언어로 묶습니다.
* **Pattern(패턴 인식)** 9/10: (i) 차분 상수/기울기, (ii) 누적합/면적, (iii) 비율/정비례가 상위 개념의 공통 골격.
* **Synthesis(종합)** 8/10: **지식그래프(JSON)** + **태그 표준** + **전이 과제 템플릿** + **LRC(Leap Ready Criteria)**로 운영.
* **Assumption, Bias(고정관념/편향)** 낮춤: “학년=상한”을 버리고, 안전장치(LRC)로 과속만 방지.

> **결론**: 구조-전이 중심으로 재저작하는 교재가 PMF를 얻기 쉽습니다. 핵심은 **문항-개념-스킬**의 3중 태깅과 “같은 구조, 다른 맥락” 묶음 설계.

### 1-2. CS(복잡성 해결)

* **하위 시스템**: (A) 개념그래프, (B) 태그 표준, (C) 아이템 뱅크, (D) 저작권·소싱, (E) 심리측정(난이도 보정), (F) 편집·디자인, (G) 파일럿 운영.
* **레버리지 포인트**

  1. **태그 표준화**: 구성요소가 재조합 가능해지고 제작 속도↑ 오류↓
  2. **전이 템플릿**: 한 개념당 3맥락 세트(생활·데이터·도형)로 확장성↑
  3. **파일럿 캘리브레이션**: Rasch/IRT로 난이도 재정렬 → 신뢰도↑
  4. **교사용 해설 구조화**: 현장 적용성↑, 재구매율↑

## 2) 교재 제작 절차(End-to-End 파이프라인)

> **전략**: *Top-Down 백캐스팅* (최상위 역량에서 내려오기) + *Bottom-Up 프로토타이핑* (초2 S1부터 빠른 파일럿) **병행**.

### 단계 0. 범위·원칙 확정 (1주)

* **아웃풋**: 교육 철학 원페이지(개념 스레드, LRC, CPA 비율), 톤&매너, 페이지 포맷.
* **결정**: 각 개념의 S1–S3 페이지 구조(예: 2–2–1 페이지), 1 유닛당 문항 수(예: 24).

### 단계 1. 개념그래프 정제 (1주)

* JSON 트리에서 **초2 진입 스레드** 우선(예: 덧셈→등차).
* 각 노드에 **선수·전이 간선** 명시, 상위 문제 예시(수능/경시 *구조*) 매핑.

### 단계 2. 태그 표준 수립 (1주)

* **필수 태그**: `concept_node, step(S1/S2/S3), micro_skill, lens(diff/acc), representation(C/P/A), context, cognitive_demand, difficulty_est, time_s, misconception, source_ref, licensing, LRC_gate`.
* **가이드북** 제작(예시·금지 사례 포함).

### 단계 3. 코퍼스 수집·분해 (2주)

* 시중 문제집, 수능, 초등 경시 **구조 분석**: 직접 복제 금지, **패턴 추출→재저작**.
* 각 문제의 **핵심 구조**(예: “차분 상수”, “등차합”, “기울기-절편”)를 서술로 기록.

### 단계 4. 아이템 뱅크 초안 (2주)

* 한 개념당 **S1(기초·조작)–S2(브리지)–S3(최상위)** 8–10–4문항 구성.
* **전이 3세트**(생활·데이터·도형)로 동일 구조 변환.
* **저작권 표기**: `source_ref`, `inspiration_only: true`, `verbatim: false`.

### 단계 5. 해설·오개념·루브릭 (병행, 2주)

* **Explain-first 해설**: “원리→사례→반례→한줄정의” 구조.
* **오개념 지도**: 예상 오답과 **교정 드릴**(1분 타임어택).
* **루브릭**: 설명형 4점 척도(핵심 용어, 구조 언급, 정확성, 일반화).

### 단계 6. 편집·페이지 디자인 (1주)

* S1은 **그림·칩·수직선** 다, S2는 표/그래프, S3는 텍스트 축약·정교한 데이터.
* **좌우 페이지 짝**: 좌(탐색)–우(전이).
* **아이콘**: lens, representation, step 표시.

### 단계 7. 파일럿 A (2주, 20~40명)

* 초2–초4 혼합 그룹. **반응시간·정답률·설명 텍스트** 수집.
* **난이도 보정**(Rasch/2PL), 문항 삭제/수정.

### 단계 8. 교사 코멘트 라운드 (1주)

* 교사 3~5인 블라인드 리뷰: “현장 난이도 체감”, “오개념 누락”, “지시문 명료성”.

### 단계 9. 파일럿 B & 수능/경시 브리지 검증 (2주)

* S3가 상위 문제(수능·경시 구조)로 **실제 전이**되는지 확인.
* **LRC 임계치** 조정(정확≥90%, 반응시간 백분위≥60 등).

### 단계 10. 1판 확정·산출물 (1주)

* **학생용 교재**(워크북) + **교사용 해설서**(수업 안, 오개념 처방, 판서 예시) + **부모 리포트 샘플**.

### 단계 11. 디지털 동시 출시(선택)

* **전이 과제 자동 생성기**(동일 구조 다른 맥락 랜덤), **미시 스킬 드릴** 탑재.

### 단계 12. 운영 데이터 루프(지속)

* 오답 패턴 히트맵 → 다음 판 수정, **아이템 뱅크 버저닝**(v1.1, v1.2…).

## 3) 태그·그래프 규격(샘플)

```json
{
  "id": "ALG-AP-S1-002",
  "title": "계단을 일정하게 올라가요",
  "concept_node": "ALG6:수열-등차",
  "step": "S1",
  "lens": ["difference"],
  "representation": "C->P",
  "context": "계단놀이",
  "micro_skills": ["더하기 누적", "차이 읽기"],
  "cognitive_demand": "이해-적용",
  "difficulty_est": 0.35,
  "time_s": 60,
  "misconception": ["차이가 매번 바뀐다고 착각"],
  "LRC_gate": "S2_open_if(acc>=0.9 && rt_pct>=0.6)",
  "source_ref": "self-authored_from_pattern",
  "licensing": "original"
}
```

**그래프 간선 예**

* `NUM1(덧셈 기초) → ALG6(등차) [type: prerequisite]`
* `ALG6(등차) → ALG4(일차함수) [type: transfer, lens: difference]`

## 4) 소싱·저작권 가이드(매우 중요)

* **원칙**: 수능/경시/시중 교재의 *문장·숫자·그림*을 그대로 복제하지 말고, **구조만 추출**해 **재저작**합니다.
* **프로세스**

  1. 원문항 → 2) *구조 서술화*(“차분 상수로 일반항 찾기”) → 3) **새 맥락·수치·표현**으로 재작성 → 4) 내부 표절검사(문장 유사도/숫자 패턴 포함).
* **메타데이터**: `source_ref`에 “영감/구조 참조” 수준 기록(내부용), 외부 공개본에는 원문항 텍스트나 고유 표기를 포함하지 않음.
* **이미지**: 도형·표는 **직접 제작**(벡터) 또는 저작권 클리어된 에셋 사용.
* **법무 체크리스트**: 라이선스 테이블(출처/권리/사용범위/증빙) 유지.

## 5) 유닛 템플릿(페이지 구성)

**Unit = [S1 기초 2p] + [S2 브리지 2p] + [S3 최상위 1p] + [리뷰 1p] = 6p**

* **S1**(초2 표현): 조작·그림, 수직선, “같은 만큼 늘어남” 언어화.
* **S2**(브리지): 표→식→그래프 전환, (a_n=a_1+(n-1)d) *말로 설명* 유도.
* **S3**(최상위): 수능·경시 **구조 동형** 문제(신규 문맥), 설명형 루브릭 포함.
* **리뷰**: 전이 3세트(생활/데이터/도형) + 오개념 교정 드릴(1분).

## 6) 샘플 유닛(요약) — “덧셈 → 등차수열”

* **S1-1(조작)**: 시작칩 2개, 매번 3개씩 더한다. 5번째는? → 표 채우며 “차이가 3” 언어화.
* **S1-2(그림→표)**: 수직선에서 +3씩 점프. 점프 횟수와 위치 관계 말로 정리.
* **S2-1(표→식)**: 시작 4, 공차 5. (a_1=4, d=5). (a_n=4+5(n-1))를 *자기 설명*으로 쓰기.
* **S2-2(그래프)**: (n, a_n) 점 찍기 → 일직선, 기울기=5 확인.
* **S3(최상위)**: “계단 면적 누적” 데이터가 등차로 증가. 10번째까지의 합을 두 방식(직접식/그래프 사다리꼴 근사)으로 비교 설명.
* **리뷰(전이)**: (i) 용돈 적립, (ii) 타일 높이, (iii) 온도 변화 데이터—세 맥락에 구조 동일 문제.

## 7) 천재적 아이디어 15선(교재 제작 특화, **3,000자+**)

1. **Isomorphic Triplets(동형 3형제)**

   * 한 핵심 구조(예: 등차 차분 상수)를 **생활–데이터–도형** 3맥락으로 동시에 제시.
   * *효과*: “문맥 의존 학습”을 차단, 전이 가속.
   * *제작법*: 아이템 템플릿에 `context_variant: ["life","table","geometry"]` 필드를 두고 자동 렌더.

2. **Lens Badge(차분/누적 배지)**

   * 각 문항 머리칸에 🔺(difference), ⬛(accumulation) 아이콘 표기.
   * *효과*: 아이가 스스로 “어느 렌즈로 볼지” 선택하는 습관 형성 → 메타인지 강화.
   * *교사용*: 렌즈 전환 질문 스크립트 제공.

3. **CPA 스위치 페어**

   * 같은 구조를 **C→P** 페이지와 **P→A** 페이지로 페어 구성(앞/뒤).
   * *효과*: 저연령 진입성과 추상 전이를 동시에 확보.
   * *제작법*: 도식화 라이브러리(수직선, 바둑판, 타일) 표준 컴포넌트화.

4. **Explain-First 해설박스**

   * 모든 S2/S3 문항에 *정답보다 먼저* “왜 그 식이 되는지” 한 줄 설명 칸 배치.
   * *효과*: 계산 선행을 억제하고 구조 언어화 습관화.
   * *루브릭*: 핵심어(차이, 기울기, 시작값) 등장 여부 체크.

5. **오개념 경고 라벨(⚠️Misconception)**

   * 페이지 하단에 빈출 오류를 미리 제시(예: 공차와 첫항 혼동).
   * *효과*: 오류 인식-회피 메타인지.
   * *교사용*: 해당 오류의 빠른 교정 드릴 3문항 묶음 제공.

6. **전이 모의고사(Structure-Only Test)**

   * 단원 말에 “같은 구조 6문항, 맥락 6종”으로 구성된 소형 모의고사.
   * *효과*: 전이 지표 수집, 난이도 캘리브레이션 데이터 확보.
   * *운영*: 시간 제한과 무제한 모드 두 가지.

7. **Anchor Items(앵커 문항) 시스템**

   * 유닛마다 2–3개 표준 문항을 고정 배치(판마다 동일) → 난이도 추적.
   * *효과*: 판(에디션) 간 난이도 동등화(수능처럼 척도 유지).
   * *데이터*: Rasch difficulty 파라미터 기록.

8. **Backwards Bridge(역방향 브리지)**

   * 고학년 개념을 먼저 스냅샷으로 보여주고(예: 일차함수 그래프), 그걸 **등차 구조로 해체**하게 하는 활동.
   * *효과*: 상·하향 전이의 양방향 강화, “낯설지만 같은 구조” 경험.

9. **Micro-Drill 60s(1분 드릴)**

   * 자리올림, 차이 읽기, 표→식 변환 등 원자 스킬을 60초 집중.
   * *효과*: 절차 자동화 → 인지 여유 확보, S3 도전 여력 상승.
   * *편집*: QR로 타이머·미니게임 연동(디지털).

10. **StoryMath 카드**

    * 초2에 맞춘 메타포 스토리(계단, 사다리, 빙고판)로 **구조 은유** 제공.
    * *효과*: 정서적 몰입 → 설명형 응답의 질 향상.
    * *운영*: 스토리→표→식으로 점진적 전환.

11. **수능·경시 구조 아틀라스**

    * 과거 수능/경시를 **구조 도식**으로만 요약한 1–2p 화보(텍스트·수치·그림은 원문 미사용).
    * *효과*: “우리가 지금 이걸 향하고 있다”를 시각화, 동기 강화.
    * *법무*: 원문 재현 금지, 구조 키워드만.

12. **Check–Mate(짝 설명) 시트**

    * 짝과 서로의 설명을 체크하는 표준 폼(핵심어, 반례, 일반화 칸).
    * *효과*: 동료 설명을 통한 이해 고착.
    * *교사용*: 5분 활동으로 수업 마무리 루틴화.

13. **스페이싱·인터리빙 달력**

    * S1/S2/S3를 섞어 복습하는 주간 달력, QR로 단기 테스트 제공.
