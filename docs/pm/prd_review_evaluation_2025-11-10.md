# 개요

본 문서는 `docs/pm/prd_review_2025-11-10.md`(이하 "리뷰")에 대한 **평가(메타 리뷰)**와 **수용/보완 계획**을 정리합니다. 목적은 리뷰의 핵심 지적을 합리적으로 수용하고, PRD v1.0.2로 구체적인 문구/구조 수정을 실행하는 것입니다.

## 총평
- 전반적으로 리뷰는 **정확한 문제 진단**(Phase 혼재, API 스펙 불완전, 커리큘럼 목록 부재, 타임라인 문구 불일치, 품질 게이트 검증 절차 미흡)을 제공하며, **실행 친화적인 제안**(부록 추가, 별도 스펙 문서, 체크리스트, 우선순위 표)을 담고 있습니다.
- 당사 현행 문서군(SSOT PRD v1.0.1, Spec, DECISIONS, SOP, Transfer Package)과 **높은 정합성**을 보이며, 개선 방향 또한 기존 의사결정 흐름과 충돌하지 않습니다.

## 리뷰 강점(우수 포인트)
1) **Phase 구분의 필요성**을 다양한 섹션에서 지적(섹션2·4·12·부록) → 독자 혼동 최소화에 실효적.
2) **API 계약 미비**를 구체적으로 제시(요청/응답/에러/인증/제약) → BE/FE 협업 리스크 감소.
3) **품질 게이트의 ‘검증 방법’**을 체크리스트화 제안 → CI/테스트 파이프라인 연계에 적합.
4) **로드맵·체크리스트 현실화**(“72시간”→“10–12주”) → 팀 속도/범위 정렬.
5) **별도 문서 체계 제안**(curriculum/api/dev) → 문서 탐색성·추적성 향상.

## 수용/보완/반박 요약
- 수용(P0):
  - “한 달”→“10–12주” 문구 일치화(마무리 섹션)
  - 섹션4에 명확한 Phase 블록(전체 비전 vs Phase 1)
  - 커리큘럼 전체 목록(12 코스·AS 목록) 부록/링크 추가
  - API 요약 스니펫(성공/오류/RFC9457) 추가 또는 별도 api 문서 링크
- 수용(P1):
  - 섹션2(JTBD)에 Phase 1/2+ 구분 박스 추가
  - 섹션6(MQG-P1)에 “검증 체크리스트 + CI 통합” 소절 추가
  - 부록B를 “10–12주 실행 체크리스트”로 교체
- 수용(P2):
  - KPI 측정 도구/주기, 법무 가이드 초안, 경쟁사 비교표 보강
- 보완/설명:
  - MQG-P1 ‘기준’은 Spec에 이미 등재됨. 리뷰 지적은 ‘검증 절차’ 미흡에 초점 → 체크리스트·CI 연동을 추가해 해소.
  - 커리큘럼/AS 목록은 Phase 1 우선 범위(코스3, AS20)를 먼저 공개하고, 나머지는 Phase 2 문서로 점진 공개.
- 반박: 없음(리뷰 주요 지적은 타당하며, 실행 가능한 범주)

## 변경 계획(PRD v1.0.2 제안)
아래 항목을 PRD v1.0.2에 반영합니다. 변경사항은 `docs/CHANGELOG.md` 및 `docs/pm/DECISIONS.md`에 기록합니다.

### A. 문구/구조 반영(P0, 3일)
1) 마무리 문구: “**한 달 내 베타**” → “**10–12주 로드맵 내 베타**”
2) 섹션4 커리큘럼:
   - “전체 비전(12 코스 × S1/S2/S3)” vs “Phase 1(코스3 × S1/S2)” **블록 분리 표기**
   - Phase 1 코스 명시: C01 덧셈, C03 곱셈, C05 분수
3) 커리큘럼/AS 목록(부록D 또는 별도 문서 링크):
   - `docs/curriculum/course_list_v1.0.md` — 12 코스 개요 + Phase1 상세
   - `docs/curriculum/atomic_skills_v1.0.md` — AS 60~80개 개요 + Phase1 필수 20개 상세
   - DAG 파일 경로 명시: `data/graph.bipartite.json`
4) API 요약(부록E 또는 별도 문서 링크):
   - 성공/오류(RFC9457) 예시, 인증 헤더, 주요 필드 설명 간단 스니펫
   - 향후 OpenAPI 경로: `docs/api/api_spec_v1.0.md`, `api/openapi.yaml`

### B. 실행/검증 반영(P1, 1주)
5) 섹션2(JTBD): Phase1/Phase2+ 타깃·JTBD를 **이중 박스**로 구분 표기
6) 섹션6(MQG-P1): “**검증 체크리스트 + CI 통합**” 소절 추가
   - 중복 방지, 범위/자릿수, 다중해 방지, 로그 스키마 → 각 검증 주체/방법/기준/CI 훅 명시
7) 부록B 교체: “**10–12주 실행 체크리스트**”(Week 1–12)로 갱신

### C. 보완 문서(P2, 2주)
8) KPI 측정 도구/주기: 제품 섹션과 연동해 측정 주기(주간/스프린트), 도구(Analytics, Amplitude 등), 보드 정의
9) 법무 가이드 초안: `docs/legal/content_license_guidelines.md`, `docs/content_license_log.csv`
10) 경쟁사 비교표: `docs/analysis/competitive_analysis_plan.md` 결과물 링크로 연결

## 리스크 및 완화
- 범위 팽창: Phase 1 공개 범위(코스3/AS20/템플릿30) **엄수**. 부록/별도 문서도 Phase 1 중심으로 공개.
- 문서 과증식: “요약(PRD) ↔ 상세(별도)”의 2계층 유지. 중복 서술 금지, 링크 일원화.
- API-구현 괴리: OpenAPI 스켈레톤을 빠르게 발행하여 BE/FE 합의 기반으로 유지.

## 결론
리뷰의 지적은 **정확**하고 **실행가능**합니다. 본 평가서는 리뷰를 **전면 수용**하고, PRD v1.0.2로의 구체적 반영 계획을 제시합니다. P0 항목을 3일 내 반영한 후, P1/P2 보강을 이어가면 PRD 품질은 “Good → Excellent(4.6/5)”로 향상될 것입니다. 변경이 완료되면 `CHANGELOG`와 `DECISIONS`에 기록하고, 팀 정렬 회의(3일 내)에서 책임과 마일스톤을 확정하겠습니다.
