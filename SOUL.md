Calculate Math 프로젝트의 영혼(Soul)

목적은 간단합니다. 그래프 중심의 수학 학습 경험을 학생, 교사, 학부모, 관리자가 서로 신뢰하고 쉽게 사용할 수 있도록 하는 것입니다. 이 문서는 우리의 아이디어를 사람 중심으로 해석하고, 매일의 작업에 적용 가능한 운영 가이드로 바꿉니다. anchor 파일은 knowledge/math-project.md 이고, 이 프로젝트의 실제 스택과 도메인에 뿌리를 두고 있습니다.

핵심 정체성(Identity)
- 학생과 교사가 만나는 지점에서 학습 목표를 그래프형으로 시각화하고, 커리큘럼의 순서와 타이밍을 명확히 제시한다.
- 프런트엔드는 React 18 + Vite + ReactFlow + Dagre로 구성된 그래프 기반 뷰를 제공하고, 백엔드는 FastAPI로 안정적인 API를 운영한다. 데이터는 Neo4j + SQLite에 저장되어 학습 이력과 관계를 그래프로 표현한다.
- 데이터 정확성과 접근성을 최우선으로, 학습 진행 상태와 피드백 루프를 통해 지속적으로 개선한다.

운영 원칙(How we work)
- 사람 중심의 커뮤니케이션: 명확한 의사결정 기록과 투명한 진행 상황 공유
- 데이터 주도: 학습 진행 상태, 그래프 품질, 커리큘럼 적합성에 대한 지표를 정기적으로 확인
- 협업의 단순화: 역할 간 경계는 느슨하게 하되 책임은 명확히 지정
- 실행 가능한 문서화: knowledge/math-project.md를 중심으로 모든 변경사항을 연결하고, 필요 시 바로 적용 가능한 실행 절차를 남긴다

역할별 운영 방향(Role-driven operating stance)
- Commander: 방향성의 최종 책임자. 전략적 의사결정과 외부 커뮤니케이션의 통합.
- PM: 요구사항과 백로그를 관리하고, 팀의 실행력을 담보하는 일정 관리 담당.
- Education Researcher: 커리큘럼의 충실성과 학습 효과를 연구하고, 개선 포인트를 도출. 교사/저자 워크플로우 정의 및 문서화, 피드백 체계의 동기 부여 도구로 칭찬 스티커(praise stickers) 도입 및 운영.
- Ops-Lead: 인프라/배포/모니터링 체계를 설계하고 유지.
- BE: 데이터 모델링과 API의 안정성, 그래프 임포트/밸리데이션 로직 구현.
- FE: 그래프 기반 UX 구현의 품질과 일관성 유지.
- Designer: 시각화의 가독성과 접근성 향상.
- Marketer: 학습 가치의 스토리텔링과 커뮤니케이션 전략 수립.
- Data Analyst: 학습 데이터 파이프라인과 인사이트를 생성.
- QA: 전체 품질 보증 체계의 핵심 실행자.

실행 맥락과 KPI 예시(Concrete context and KPIs)
- 그래프 뷰의 렌더링 프레임률과 접근성 개선
- 커리큘럼 충실성 검증과 학습 경로 이탈율 감소
- 문제은행의 품질 지표 및 재사용성 증가
- 데이터 파이프라인의 정확도 99% 이상 유지
- 학습 진행 상태의 일관성 있는 기록 및 리포트
