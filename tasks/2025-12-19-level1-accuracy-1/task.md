# Level 1 스킬 문제풀이 & 정답률 계산 — MVP v0.1

## 목표(Goal)
- Level 1 스킬/문제 데이터를 JSON으로 관리하고 목록/문제 수를 안정적으로 노출한다.
- 스킬 선택 → 객관식 단일선택 풀이 → 정오 판정 → 정답률 계산까지 최소 루프를 완성한다.
- 데이터 꼬임 방지 규칙(중복/누락/양방향 매핑)을 강제한다.
- (옵션) 세션/누적 정답률을 분리하고 로컬에 보존한다.

## 작업 단위(Definition of Done)
- (필수) Skill/Problem/Attempt 스키마 확정 + 샘플 데이터(스킬 2~3개, 문제 6~10개) 제공.
- (필수) 데이터 검증 로직 구현: unique ID, 존재 참조, choices/answer_index 범위, 양방향 매핑 불일치 오류.
- (필수) API 또는 프론트 로직으로 스킬 목록/문제 세트/정오 판정/정답률 산출이 동작한다.
- (필수) UI 3화면(스킬 리스트/문제 풀이/결과)에서 정답 수·총 문항·정답률을 표시한다.
- (권장) 로컬 저장으로 최근 기록 유지, “이번 세션” vs “누적” 정답률 분리 표시.
- (권장) 테스트 추가: 검증 실패 케이스와 정답률 계산 일치 검증.

## 범위(Scope)
- Level 1 스킬만 다룸; 스킬 리스트는 `skill_id/title/description/problem_ids` 기반으로 렌더링.
- 문제 형식은 객관식 단일선택, 고정 순서 제공(랜덤/가변 없음).
- 정답률 계산: accuracy = (정답 수 / 시도 수) * 100, 최소 표기(정답/총/%) 제공.
- 데이터 소스는 JSON 우선(사람이 직접 수정 가능), 로컬 기반 로딩을 기본으로 가정.
- 오프라인 동작 가능해야 하며, 데이터 로딩 실패 시 어떤 ID가 문제인지 메시지 노출.
- 성능 목표: 문제 300개 기준 리스트/풀이 흐름이 버벅이지 않음.

## 비범위(Non-goals)
- Level 2 이상 스킬/선수관계, 스킬트리 시각화(복잡한 그래프)
- AI 문제 생성/튜터링/해설 자동화, 추천/적응형 난이도
- 주관식 채점, 오답노트, 힌트 제공
- 로그인/계정/클라우드 동기화
- 문제 랜덤 출제, 문제 편집 UI, 외부 스프레드시트 연동

## 결정 필요(질문)
- 문제 형식은 객관식 단일선택이 맞는가?
- 정답 공개는 제출 즉시 vs 종료 후 중 무엇인가?
- 정답률 범위는 단일 스킬만 vs 다중 스킬 통합 세션도 필요한가?
- 데이터 입력/저장 위치는 JSON 파일/DB/외부 시트 중 무엇인가?
- 문제 순서는 고정 vs 랜덤(랜덤은 v0.2로 미룸 가능)?
- 대상 플랫폼은 웹(React) 기준으로 확정인가?

## 참고
- 권장 스키마: Skill/Problem/Attempt 구조는 PRD v0.1의 JSON 예시를 따른다.
- 데이터 검증 규칙은 최소 7개(양방향 매핑 포함)를 모두 적용한다.
- 세션/누적 분리는 v0.1 범위를 넘지 않는 “+1 옵션”으로 취급한다.
