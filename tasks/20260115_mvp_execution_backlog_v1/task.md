# MVP 실행 백로그 (Done/Partial/To-do 기준)

## Goal
현재 구현된 기능을 기준으로 **Done/Partial/To-do**를 확정하고, MVP 완성을 위한 실행 순서를 정립한다.

---

## Current Status
- `curriculum-viewer` 프론트엔드 존재
- 트리/그래프/리포트 네비게이션 존재
- 상세 패널 존재
- 즉시 채점 방식 (일괄 제출로 변경 필요)
- 학습 진행 상태 표시 부재 (리뷰어 지적)
- 전체 학습 현황 대시보드 부재 (핵심 이슈)

---

## 상태 표기 규칙
- **DONE**: 리뷰어 피드백상 이미 확인된 기능
- **PARTIAL**: 기능은 있으나 MVP 요구사항과 다름 (수정 필요)
- **TO-DO**: 리뷰어가 "없다"고 지적했거나 필수인데 흔적이 없는 것
- **VERIFY**: Audit로 확정해야 하는 것

---

## Requirements

### EPIC 0 — 레포 실사(Audit) + MVP 기준 재정렬 (P0)

> 이걸 먼저 해야 "꼬임"이 사라진다. PM/개발자가 1~2시간 안에 끝낼 수 있는 수준.

#### MVP-A00 현재 구현 기능 인벤토리 체크리스트 작성
- **상태**: TO-DO
- **담당**: FE+PM
- **작업**:
  - FE 실행: 라우트 목록, 페이지 스크린샷 저장
  - BE 실행: API 목록 캡처
  - 진행 저장 위치 확인 (LocalStorage/DB/파일)
- **DoD**: "기능 목록 + 구현 위치 + 스샷" 문서 1개 생성

#### MVP-A01 '즉시 채점' → '일괄 제출 채점' 변경 영향도 분석
- **상태**: TO-DO
- **담당**: FE
- **작업**: 채점 위치, 제출 데이터 구조, 결과 저장 구조 정리
- **DoD**: 변경 범위 문서화

#### MVP-A02 "학생 모드 vs Author 모드" 화면/권한/라우트 정리
- **상태**: TO-DO
- **담당**: FE+PM
- **DoD**: Author 전용 기능과 Student 기능 분리 표 완성

## 실행 순서 (P0만)

1. **MVP-A00/A01/A02** — Audit로 Done/Partial 확정
2. **MVP-090, MVP-073** — 진행 데이터(노드 상태) 구축
3. **MVP-0303/0304/0305** — 지도 상태 시각화 + 잠김 이유
4. **MVP-0404/0406 + 0403** — 즉시 채점 → 일괄 제출 채점 전환
5. **MVP-0502/0504/0505** — 평가 페이지 강화 + 다음 스텝 이동
6. **MVP-0601/0602** — 대시보드 추가 + 네비 메뉴
