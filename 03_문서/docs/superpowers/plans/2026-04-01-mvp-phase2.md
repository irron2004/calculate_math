# MVP Phase 2: AtomicSkill DAG + Self-Diagnosis

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add AtomicSkill nodes to the learning graph, tag all problems with required skills, and build a self-diagnosis UI on the evaluation screen so students can identify why they got answers wrong.

**Architecture:** The learning graph JSON gains 7 AtomicSkill (AS) nodes and two new edge types — `teaches` (CS→AS: completing this unit grows this skill) and `requires_skill` (AS→CS: this skill is needed to start this unit). Each problem in the problem bank gets a `required_skills` array. After a student submits with wrong answers, EvalPage shows a self-diagnosis section: options are the union of required_skills from wrong problems plus a fixed "계산 실수예요" choice. The chosen diagnosis is POSTed to a new backend endpoint and stored in `study_sessions.diagnosis_json`.

**Tech Stack:** FastAPI/SQLite backend, React 18/TypeScript frontend, existing `authFetch`, existing `loadProblemBank` for problem loading, existing `GradingResultV1.perProblem` for wrong-answer detection.

## AtomicSkill Reference

| ID | Label |
|----|-------|
| `AS.NUMBER_SENSE` | 수 세기와 수의 크기 비교 |
| `AS.PLACE_VALUE` | 자릿값 이해 (일, 십, 백, 천) |
| `AS.ADD_SUB` | 덧셈과 뺄셈 |
| `AS.MUL_DIV` | 곱셈과 나눗셈 |
| `AS.FRAC_BASIC` | 분수 개념과 크기 비교 |
| `AS.DECIMAL` | 소수 개념과 자릿값 |
| `AS.RATIO` | 비율과 비례 |

Special constant (always shown in diagnosis, NOT a graph node):
- `CALC_MISTAKE` / label: `계산 실수예요`

```typescript
import { authFetch } from '../auth/api'
import type { DiagnosisChoice } from './types'

const API_BASE = import.meta.env.VITE_API_URL || '/api'

export async function saveDiagnosis(
  sessionId: string,
  choice: DiagnosisChoice
): Promise<void> {
  try {
    const res = await authFetch(
      `${API_BASE}/study-sessions/${encodeURIComponent(sessionId)}/diagnosis`,
      {
        method: 'PATCH',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ skillId: choice.skillId, label: choice.label })
      }
    )
    if (!res.ok) return
  } catch {
    // Best-effort — diagnosis failures don't block the user
  }
}
```
