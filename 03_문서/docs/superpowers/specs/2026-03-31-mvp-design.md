## 1. 한 줄 정의

**"왜 이 단원인지, 왜 틀렸는지를 학생 스스로 이해하게 만드는 수학 학습 서비스"**

단순 문제 풀기가 아닌, **설명 가능한 추천**과 **오답 원인 자기 진단**을 통해 메타인지를 키운다.

---

## 2. 타겟 & 사용 맥락

| 항목 | 내용 |
|---|---|
| 주 사용자 | 초등학교 1~6학년 학생 |
| 결제자 | 학생 본인 또는 학부모 (B2C 개인) |
| 사용 상황 | ① 스스로 공부하고 싶을 때 / ② 오늘 틀린 문제를 복습할 때 |
| 수익화 | MVP 단계에서는 전체 무료 |

## 3. 서비스 핵심 가치

> "무엇을 추천할지"보다 **"왜 이 추천인지, 왜 틀렸는지"를 설득력 있게 보여주는 것**이 승부처.

두 가지 핵심 엔진:

1. **단원 그래프 (Learning Path Graph)**
   - 노드: CourseStep(단원) + AtomicSkill(세부 스킬)
   - 엣지: `requires`(선수 필요) / `teaches`(이 단원이 키워주는 스킬) / `enables`(다음 추천 경로)
   - 역할: 다음에 할 단원 결정 + "왜 이 단원인지" 근거 생성

2. **오답 원인 진단 (Error Cause Diagnosis)**
   - 오답 직후 학생이 직접 원인 선택 (자기 진단)
   - 선택지: 이 문제에 필요한 스킬 목록 + "계산 실수"
   - 역할: 선택한 원인 → 재학습 단원 자동 연결

## 4. 스킬 트리 구조 (이중 레이어 DAG)

### 레이어 구성

```
CourseStep (CS) — 단원 노드
  예: C01-S1 덧셈 기초, C03-S1 구구단, C05-S1 분수 개념

AtomicSkill (AS) — 세부 스킬 노드
  예: AS.ADD.BASIC, AS.MUL.FACTS, AS.FRAC.CONCEPT
```

### 엣지 타입

| 타입 | 방향 | 의미 |
|---|---|---|
| `requires` | AS → CS | 이 단원을 시작하려면 필요한 스킬 (최소 레벨 명시) |
| `teaches` | CS → AS | 이 단원을 완료하면 올라가는 스킬 |
| `enables` | CS → CS | 이 단원 완료 후 추천되는 다음 단원 |

### Phase 1 범위 (초1~6)

- **CourseStep**: 덧셈 / 뺄셈 / 곱셈 / 나눗셈 / 분수 / 소수 × S1/S2 (총 12 노드 내외)
- **AtomicSkill**: 약 20~30개 핵심 스킬

## 5. 핵심 화면 & 플로우

### 5-1. 첫 화면 (홈)

학생이 앱을 켰을 때 가장 먼저 보는 화면. 두 섹션으로 구성:

**① 오늘의 추천 (1~2개)**
```
╔══════════════════════════════╗
║  ✨ 지금 이거 해봐!           ║
║                              ║
║  분수의 덧셈                  ║
║  → 지난번에 3번 틀렸어요      ║  ← 한 줄 이유
║  [시작하기]                   ║
╚══════════════════════════════╝
```

추천 근거 유형:
- "지난 세션에서 N번 틀렸어요"
- "선수 스킬이 모두 준비됐어요"
- "오랫동안 안 풀었어요"

**② 오답 복습 (최근 3~5개)**
```
╔══════════════════════════════╗
║  🔥 다시 도전해봐!            ║
║                              ║
║  • 분수의 뺄셈  틀린 문제 2개 ║
║  • 두 자리 곱셈 틀린 문제 1개 ║
╚══════════════════════════════╝
```

### 5-2. 문제 풀기 화면 (세션 플레이어)

- 문제 표시 + 입력
- 진행률 표시 (N/5 문제)
- [제출] / [모르겠어요(스킵)]

### 5-3. 오답 직후 화면 (자기 진단)

오답 시 바로 표시:
```
왜 틀렸을까요?

○ 분수 통분을 몰랐어요        ← 이 문제의 필요 스킬
○ 분자끼리 더하는 법을 몰랐어요 ← 이 문제의 필요 스킬
○ 계산 실수예요               ← 항상 포함
```

선택 후 → 해당 스킬 재학습 단원 연결

### 5-4. 세션 결과 화면

- 정답률 (N/5)
- AS 스킬 레벨 변화 표시
- 완료 조건 충족 시: "단원 완료! ⭐" + 새로 열린 단원 표시
- 미충족 시: "조금 더 연습해볼까요?" + 다시 도전 버튼

### 5-5. 스킬 트리 지도 (탐색용)

- CourseStep 노드: 완료/진행중/가능/잠김 상태
- 노드 클릭 → "이 단원에 필요한 스킬 / 이 단원이 열어주는 단원" 표시
- 현재 구현된 StudentStageMapPage 개선

### 5-6. 온보딩 (신규 가입)

1. 학년 선택
2. 짧은 진단 테스트 (5~10문제)
3. 시작점 스킬 트리 자동 설정 → 홈으로 이동

## 7. 데이터 저장

### 변경 사항
- 현재: 브라우저 로컬 스토리지
- MVP: **서버 저장** (기기 바뀌어도 이어짐)

### 핵심 저장 데이터

| 데이터 | 내용 |
|---|---|
| 세션 기록 | 풀이 일시, 단원 ID, 시도한 문제, 정/오답, 소요 시간 |
| 오답 진단 | 문제 ID, 학생이 선택한 원인 (스킬 ID or "계산실수") |
| AS 레벨 | 스킬 ID별 현재 레벨 (0~3), 누적 XP |
| 추천 로그 | 추천된 단원 ID, 추천 근거, 클릭 여부 |

## 8. 문제 콘텐츠 전략

| 문제 유형 | 방식 |
|---|---|
| 연산 드릴 | 자동 생성 (op / digits / count / seed 파라미터) |
| 개념 문제 | 관리자 직접 저작 + 필요 스킬 태깅 |

### 문제에 달아야 하는 태그 (저작 시)
- 관련 CourseStep ID
- 필요 AtomicSkill 목록 (오답 진단 선택지 생성용)
- 렌즈 유형 (선택): 차분 / 누적 / 비율 / 스케일

---

## 9. 추천 엔진 로직 (MVP)

```
추천 점수 = w1 × 그래프 가능성 + w2 × 오답 빈도 + w3 × 마지막 시도 경과일
```

- **그래프 가능성**: `requires` 충족 여부 (충족 안 하면 0)
- **오답 빈도**: 최근 2주 내 해당 단원/스킬 오답 횟수
- **마지막 시도 경과일**: 오래 안 한 단원 보정

추천 결과: 최대 2개, 각 추천에 한 줄 근거 텍스트 포함

## 10. 개발 접근 방식 — 접근 B (얇은 슬라이스)

> 빠르게 학생에게 보여주면서 검증하고, 이중 레이어 DAG를 단계적으로 붙인다.

### 단계 1 (2~3주): 홈 화면 + 서버 저장
- 홈 화면 개편 (추천 + 오답 복습)
- 세션/시도 서버 저장 전환
- 단순 추천 로직 (오답 빈도 기반)

### 단계 2 (3~4주): DAG + 진단
- CourseStep + AtomicSkill 데이터 정의
- 오답 직후 자기 진단 UI
- 추천 근거 생성 (그래프 기반)

### 단계 3 (2~3주): 완성도
- 온보딩 (학년 선택 + 진단)
- 세션 결과 화면 (스킬 레벨 변화 표시)
- 스킬 트리 지도 개선
