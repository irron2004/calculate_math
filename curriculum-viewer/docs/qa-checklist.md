# QA 체크리스트 - test1 계정 로그인 후 화면 검증

이 문서는 Curriculum Viewer 애플리케이션의 QA 검증 절차를 정리합니다.

## 사전 준비

### 1. 서버 실행
```bash
cd curriculum-viewer
npm run dev
```

서버가 실행되면 다음 주소로 접속:
- `http://localhost:5173/` (WSL2 자동 포트 포워딩)
- `http://172.23.182.109:5173/` (WSL IP 직접 접속)

### 2. 테스트 계정 생성
test1 계정이 없으면 회원가입이 필요합니다:
- 아이디: `test1`
- 비밀번호: `test1`
- 이름: `test1` (또는 원하는 이름)
- 학년: `3` (또는 원하는 학년)
- 이메일: `test1@test.com` (유효한 이메일 형식)

## 발견된 이슈 및 개선 사항

### 알려진 이슈
1. **test1 계정 자동 생성 없음**: test1 계정은 회원가입으로 생성해야 합니다.
2. **초기 데이터 없음**: 새 계정은 학습 이력이 없어 리포트가 비어 있을 수 있습니다.
3. **그래프 데이터 의존성**: 학습 그래프 데이터(`learning_graph_v1.json`)가 없으면 일부 기능이 동작하지 않을 수 있습니다.

### 개선 제안
1. **테스트 계정 자동 생성**: 개발 환경에서 테스트 계정 자동 생성 기능 추가
2. **샘플 데이터**: 새 사용자를 위한 샘플 학습 데이터 제공
3. **에러 처리 개선**: 더 명확한 에러 메시지 및 복구 방법 제시
4. **성능 최적화**: 큰 그래프 렌더링 성능 개선
